# How the Frontend Was Built

## What is a Frontend?
Think of a smart contract like a vending machine — it does the work, but you need buttons to interact with it. The frontend is those buttons. It's the website you see and click on.

---

## What tools did we use and why?

### React
React is a JavaScript library for building websites. Instead of writing one giant messy file, React lets you break your page into small reusable pieces called **components**. Think of it like LEGO blocks — each block does one thing, and you snap them together to build the full page.

We used React because:
- It keeps the code organised and clean
- Each section of the page (wallet button, mint buttons, balances) lives in its own file
- It automatically updates the page when something changes (like your balance after minting)

### Vite
Vite is the tool that sets up and runs the React app on your computer. Think of it like the oven that bakes the cake — you don't eat the oven, but you need it to make the cake. It starts a local development server so you can open the app in your browser at `http://localhost:5173`.

### Ethers.js
Ethers.js is a JavaScript library that lets your website talk to the blockchain. Without it, your React app has no idea what a wallet or a smart contract is. It handles:
- Connecting to MetaMask (your crypto wallet)
- Sending transactions to the contract (minting)
- Reading data from the contract (balances, total supply)

Think of it as the translator between your website and the blockchain.

### MetaMask
MetaMask is a browser extension that acts as your crypto wallet. When the website wants to mint an NFT, it asks MetaMask to sign and send the transaction. You don't code MetaMask — it just needs to be installed in your browser.

## How the files are structured

```
frontend/src/
├── abi.json          — The contract's instruction manual (auto-generated by Hardhat)
├── useContract.js    — Handles connecting your wallet and loading the contract
├── MintPanel.jsx     — The mint buttons (Mint Rose, Mint Lily, Mint Both)
└── App.jsx           — The main page that puts everything together
```

### Why split into multiple files?
Imagine writing a whole book in one sentence — it would be impossible to read. Splitting into files means:
- `useContract.js` only worries about the wallet
- `MintPanel.jsx` only worries about minting
- `App.jsx` only worries about the layout and balances

This is called **separation of concerns** — each file has one job.

---

## How it all works together

1. You open the app in your browser
2. You click **Connect Wallet** — `useContract.js` asks MetaMask for your address and loads the contract
3. You click **Refresh Balances** — `App.jsx` asks the contract how many ROSE and LILY tokens you own
4. You enter an amount and click a mint button — `MintPanel.jsx` sends the transaction to the blockchain via MetaMask
5. After the transaction confirms, your balances automatically refresh

## How to run it

1. Deploy your contract and copy the address:
```bash
npx hardhat run scripts/deploy.js --network localhost
```

2. Paste the address into `/home/wtc/Documents/ABC/SecureNFT/frontend/src/useContract.js` line 5:
```js
const CONTRACT_ADDRESS = "0xYourAddressHere";
```

3. Start the frontend:
```bash
cd frontend
npm run dev
```

4. Open your browser at `http://localhost:5173`
